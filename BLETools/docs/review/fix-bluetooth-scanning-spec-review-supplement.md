## 二、本次补充发现的新问题

### 2.1 架构设计缺失 🆕

| 序号 | 问题 | 严重程度 | 说明 |
|------|------|----------|------|
| A1 | 无架构图 | 中 | 应包含BLE扫描模块的组件关系图 |
| A2 | 无序列图 | 中 | 用户点击扫描 → 设备显示的完整流程 |
| A3 | 类/模块职责不清晰 | 中 | 未说明 `MainActivity`、`BleCallback`、`BluetoothLeScannerCompat` 的交互关系 |

**建议补充**:
```
┌─────────────┐     ┌──────────────────────┐     ┌──────────────┐
│ MainActivity │────▶│ BluetoothLeScanner  │────▶│ Device List  │
│   (View)    │     │      Compat         │     │    (UI)      │
└─────────────┘     └──────────────────────┘     └──────────────┘
       │                    │
       │                    ▼
       │            ┌───────────────┐
       └───────────▶│  ScanCallback │
                    │   (Nordic)    │
                    └───────────────┘
```

### 2.2 需求完整性问题 🆕

| 序号 | 缺失需求 | 说明 |
|------|----------|------|
| R1 | 扫描超时机制 | 扫描持续多长时间？超时后是否自动停止？ |
| R2 | 设备数量上限 | 列表最多显示多少设备？达到上限如何处理？ |
| R3 | 设备去重逻辑 | 同一设备多次扫描到，如何更新而非重复添加？ |
| R4 | 扫描功耗要求 | 扫描对电池的影响，是否有节电策略？ |
| R5 | 并发控制 | 多次点击扫描按钮如何处理？是否允许同时多次扫描？ |
| R6 | 库初始化失败处理 | `BluetoothLeScannerCompat.getScanner()` 返回 null 怎么办？ |
